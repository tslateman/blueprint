# Blueprint

Blueprint is a framework for enforcing structured intent and dynamic schemas in LLM interactions. It uses a YAML-based specification to compile system prompts, Pydantic schemas, and execution logic, integrated with long-term memory and secure sandboxing.

## Core Pillars

1.  **Spec-Driven Intent**: Define agent behavior and constraints in YAML.
2.  **Dynamic Enforcement**: Compile specs into Pydantic models via `instructor`.
3.  **Secure Execution (Vault)**: Run generated code in a Zero-Trust sandbox using `shuru`.
4.  **Long-Term Memory (Lore)**: Hydrate prompts with project-specific context via `lore`.
5.  **Continuous Evals**: A built-in semantic evaluation harness to prevent regressions.

## Getting Started

### Prerequisites

- [uv](https://github.com/astral-sh/uv)
- [just](https://github.com/casey/just)
- [shuru](https://shuru.io) & [lore](https://lore.sh) CLIs

### Setup

1. Clone the repo and install dependencies:
   ```bash
   just install
   ```
2. Configure your `.env` file with at least one API key:
   ```bash
   GEMINI_API_KEY=your_key
   # Optional fallback keys
   ANTHROPIC_API_KEY=your_key
   OPENAI_API_KEY=your_key
   ```

## Architecture & Features

### 1. Blueprint Specification (`.yaml`)

Blueprints define the agent's DNA.

**Example (`demo_spec.yaml`):**

```yaml
intent: "You are a data extractor."
constraints: ["Be concise."]
lore_context: ["architectural decisions"]
tools_allowed: ["lore_search", "lore_record_decision"]
output_schema:
  target_name: { type: "string", description: "Name of the entity" }
  confidence: { type: "integer", description: "1-100 score" }
```

### 2. Lore Integration (Memory)

If `lore_context` is defined, the `BlueprintCompiler` automatically:

1. Calls `lore recall <query>` for each item.
2. Hydrates the System Prompt with the retrieved context.
3. Injects tool documentation if `tools_allowed` are present.

### 3. Shuru Vault (Sandbox)

The `ToolRouter` executes Python code generated by the LLM inside a `shuru` container. This ensures that any "calculation" or "scraping" code cannot harm your host system.

### 4. Semantic Evaluations

Blueprint treats "AI prompts as software." We use a "Golden Case" model to verify behavior.

- **Run Evals (via Gemini CLI)**: `just eval` (Default, uses the headless CLI engine)
- **Run Evals (via Raw API)**: `just eval-api`

**Adding a Test Case (`evals/golden_cases.yaml`):**

```yaml
- id: "extraction_test"
  spec_path: "demo_spec.yaml"
  input: "I am John Doe from NYC."
  assertions:
    - type: "field_matches"
      field: "target_name"
      value: "John Doe"
    - type: "semantic"
      prompt: "The output should accurately reflect the user's origin."
```
